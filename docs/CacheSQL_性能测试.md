## 四、性能总结

| 场景 | 典型QPS | 推荐用法 |
|------|---------|---------|
| **嵌入式 get() 单线程** | 60~97万 | 最高性能，适合内部调用 |
| **嵌入式 get() 多线程** | 300~400万 | JDK 17+ 并发最优 |
| **嵌入式 SQL 单线程** | 15~31万 | SQL接口，对应用透明 |
| **嵌入式 SQL 多线程** | 75~145万 | 生产推荐，SQL透明+高性能 |
| **HTTP SQL** | 3K~17K | 运维/跨语言场景，非主路径 |

---

## 五、JDK版本建议

| JDK版本 | 综合评价 |
|---------|---------|
| **JDK 8** | 单线程get()最快(97万)，但多线程弱(68万)，SQL解析慢 |
| **JDK 11** | 多线程get()优秀(326万)，SQL性能好(27万)，推荐 |
| **JDK 17** | **综合最优**，多线程get() 357万，SQL 31万，并发扩展最好 |
| **JDK 21** | 多线程get()最强(395万)，SQL 31万，但虚拟线程等新特性暂未利用 |

**推荐**: JDK 17（综合性能最优，LTS长期支持）  
**如果只用单线程**: JDK 8 或 JDK 21 均可（get() 接近100万QPS）

## 七、测试执行详情

### 测试命令

```bash
# 单元测试
set JAVA_HOME=D:\Program Files\Java\jdk-17
mvn test -q

# 真实数据benchmark（100万行KCA2）
mvn exec:java -Dexec.mainClass=com.browise.BenchmarkTest -Dexec.classpathScope=test

# 合成数据benchmark（含HTTP）
mvn exec:java -Dexec.mainClass=com.browise.FullBenchmark -Dexec.classpathScope=test
```

### 测试配置

- config.properties: server.mode=embed, Oracle host:1521:orcl
- BenchmarkTest: 100,000次单线程迭代, 16线程×6,250次并发迭代
- FullBenchmark: 50,000次单线程, HTTP 5,000次(1线程)/16,000次(8线程)
- JVM参数: 默认(-Xmx由系统自动分配, 约7~8GB)
